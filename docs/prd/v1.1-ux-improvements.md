# v1.1 UX Improvements — 7 Features

## Status
Approved

## Overview

7 个小型 feature 改善 claudeD 的日常可用性。解决 v1.0 中的核心 UX 痛点：每条消息都触发 Claude、无法中断、无法切换模型、缺乏运维可见性。

## Motivation

v1.0 能跑，但日常使用有明显摩擦：
1. 在已绑定 channel 里任何消息都会开 thread + 启动 Claude session，想聊天都不行
2. Claude 失控时没有中断手段
3. 想用 opus 处理复杂任务时无法在 thread 里切换
4. 没有项目级 system prompt，每次都要重复上下文
5. Claude 的思考过程不可见
6. 不知道有多少 session 在跑
7. 没有健康检查入口

## Features

### F1 — @bot 触发模式

**现状**：channel 里每条消息都触发 Claude，创建 thread。
**改为**：只有 @claudeD 或以特定前缀开头的消息才触发。

**实现**：
- `on_message` 中检查：消息是否 mention 了 bot（`bot.user in message.mentions`）
- Thread 内消息不需要 @，直接转发给 Claude（thread 已经是一个 session）
- Channel 里不 @ bot 的消息静默忽略

**接口**：无新命令，行为变更。

### F2 — /interrupt

**功能**：中断正在执行的 Claude 操作。

**实现**：
- `/interrupt` slash command，必须在 thread 内使用
- 调用 `ClaudeBridge` 暴露的 `interrupt()` → SDK 的 `client.interrupt()`
- 在 thread 发送 `⚠️ Claude interrupted by user`

**接口**：`/interrupt`

### F3 — /model 切换

**功能**：在 thread 内切换 Claude 模型。

**实现**：
- `/model <name>` slash command
- 支持：sonnet, opus, haiku（或任意 model id）
- 切换时停止当前 session，用新 model 创建新 session
- 通知用户：`🔄 Switched to <model>. New session started.`

**接口**：`/model <name>`

### F6 — System prompt 自定义

**功能**：给项目设置 system prompt，每个 session 自动带上。

**实现**：
- `/project system-prompt <text>` — 设置（存入 projects.json）
- `/project system-prompt` — 查看当前
- `/project system-prompt clear` — 清除
- `ClaudeBridge.start()` 时传 `append_system_prompt` 参数

**接口**：`/project system-prompt [text|clear]`

### F8 — Thinking 折叠展示

**功能**：Claude 的 ThinkingBlock 在 Discord 用 spoiler 展示。

**实现**：
- `DiscordRenderer` 处理 `ThinkingBlock` 时：
  - 发送 `💭 ||<thinking content>||`（Discord spoiler 语法）
  - 长 thinking 截断到 1900 字符

**接口**：无新命令，自动展示。

### F9 — /session list

**功能**：列出所有活跃 session。

**实现**：
- `/session list` slash command
- 显示 embed：每行一个 session，包含 thread 名、model、cost、运行时间
- 从 `SessionManager.list_sessions()` 获取

**接口**：`/session list`

### F10 — /health

**功能**：Bot 健康检查。

**实现**：
- `/health` slash command
- 显示 embed：
  - Bot 运行时间（uptime）
  - 活跃 session 数
  - Claude CLI 版本（`claude --version`）
  - Python 版本
  - 已绑定项目数

**接口**：`/health`

## Acceptance Criteria

- [ ] AC1: 在已绑定 channel 发普通消息不会触发 Claude；@bot 后触发
- [ ] AC2: Thread 内消息仍然直接转发给 Claude（不需要 @）
- [ ] AC3: `/interrupt` 能中断正在运行的 Claude 任务
- [ ] AC4: `/model opus` 切换后新消息用 opus 回复
- [ ] AC5: `/project system-prompt "you are a Go expert"` 设置后新 session 带上该 prompt
- [ ] AC6: Claude 思考内容以 spoiler 形式展示
- [ ] AC7: `/session list` 显示所有活跃 session
- [ ] AC8: `/health` 显示 bot 状态信息

## Technical Approach

所有 feature 都是在现有模块上的增量修改：
- F1: `bot.py` on_message 加 mention 检查（~10 行）
- F2: `claude_bridge.py` 加 `interrupt()` + `bot.py` 注册命令（~20 行）
- F3: `bot.py` 注册 `/model` + 停止/重建 session（~30 行）
- F6: `project_manager.py` 扩展 + `claude_bridge.py` 传 append_system_prompt（~40 行）
- F8: `discord_renderer.py` 处理 ThinkingBlock（~10 行）
- F9: `bot.py` 注册命令 + embed 渲染（~30 行）
- F10: `bot.py` 注册命令 + embed 渲染（~30 行）

总新增约 170 行。

## Testing Strategy

- 单元测试：mention 检测逻辑、system prompt 持久化
- 手动测试：Discord 端到端验证

## Out of Scope

- Cost tracking（v1.2）
- Session resume（v1.2）
- Image 支持（v1.2）
- File diff 展示（v1.2）
- Claude 创建 channel（v1.3）
