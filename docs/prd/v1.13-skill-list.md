# PRD ÔÇö `/skill list` slash command (#109)

**Status:** Approved (manager-confirmed decisions 2026-05-11).
**Branch:** `feat/v1.13-skill-list`.
**Scope:** single PR, no subtask split. Estimated ~300 LOC including tests.

## Goal

Add `/skill list` so Discord users can discover what skills Claude will auto-inject in the current channel's session, **without** having to invoke them first.

## User-visible behavior

A new ephemeral embed-only command:

```
/skill list
```

Returns an ephemeral embed visible only to the invoking user, grouped by source.

### Happy path (bound channel, active session)

Title: `­čž░ Skills (N)` where N = count of skill-backed slash commands.

Body grouped by:
1. **Project** ÔÇö commands whose description ends with `" (project)"`
2. **User (Global)** ÔÇö commands whose description ends with `" (user)"`
3. **Plugin: <name>** ÔÇö commands whose description ends with `" (plugin:<name>)"` (one section per plugin, alphabetized)

Each row: `ÔÇó <name> ÔÇö <description with source-tag suffix stripped, truncated to 120 chars + ÔÇŽ>`

Built-in commands (`clear`, `compact`, `init`, `review`, ÔÇŽ) are **excluded** ÔÇö they have no source-tag suffix and are not user-installed skills.

### Edge cases

- **Unbound channel** ÔÇö STILL WORKS. The temp-client path uses `setting_sources=["user"]` only (no `"project"` / no `"local"`). Project section is omitted from the embed; user is told via a footer line: `_­čĺí Unbound channel ÔÇö showing global skills only. Run `/project bind <path>` to see project skills._`
- **Bound but no active session** ÔÇö defer + spin up a temporary `ClaudeSDKClient`. UX shows "Loading skillsÔÇŽ" via `interaction.response.defer(ephemeral=True)`. Latency budget 5 s; CLI cold start is ~2-4 s in practice.
- **CLI not found / connection error** ÔÇö catch `CLINotFoundError`, `CLIConnectionError`, generic `Exception`. Return a friendly red embed titled `ÔŁî Skills unavailable` with the error class name in the description (not full stack).
- **`get_server_info()` returns `None`** ÔÇö same friendly error path.
- **No skills found** (only built-in commands) ÔÇö embed shows `­čž░ Skills (0)` + a body line `No user or project skills installed.`
- **>4000 chars total embed content** ÔÇö truncate with `ÔÇŽand N more skills (use Claude CLI to see all)`. We don't paginate in v1.13; live probe currently returns 2 user skills, so the truncation branch is mostly defensive.
- **Half-broken SKILL.md** (description is `""` from CLI) ÔÇö display as `ÔÇó <name> ÔÇö _(no description)_`.

## Architecture

New file: `src/clauded/cogs/skill.py`.

Pattern: mirrors `cogs/agent.py` (single-file group + decorators, no manager class).

### Skill resolution priority

```
Path A ÔÇö active bridge piggyback:
    bridge = session_manager.get_session(thread_id_or_channel_id)
    if bridge and bridge.is_active and bridge._client is not None:
        info = await bridge._client.get_server_info()
        # 0-1 ms read of cached _initialization_result

Path B ÔÇö fallback (no active session, or bridge has no client):
    cwd, is_bound = project_manager.get_path_or_default(channel_id)
    setting_sources = ["user", "project", "local"] if is_bound else ["user"]
    async with ClaudeSDKClient(ClaudeAgentOptions(cwd=str(cwd), setting_sources=setting_sources)) as tmp:
        info = await tmp.get_server_info()
    # ~2-4 s cold start
```

Skill filtering:
```python
def _classify(cmd: dict) -> tuple[str, str, str]:
    """Return (group, displayName, displayDescription) or ('', '', '') for built-ins."""
    name = cmd.get("name", "")
    desc = cmd.get("description", "") or ""
    if desc.endswith(" (user)"):
        return ("user", name, desc[:-7].rstrip())
    if desc.endswith(" (project)"):
        return ("project", name, desc[:-10].rstrip())
    if desc.endswith(")") and " (plugin:" in desc:
        # " (plugin:foo)" ÔÇö extract plugin name
        idx = desc.rfind(" (plugin:")
        plugin_name = desc[idx+9:-1]
        return (f"plugin:{plugin_name}", name, desc[:idx].rstrip())
    return ("", "", "")  # built-in ÔÇö filter out
```

Channel resolution (same pattern as `_unbound.reject_if_unbound`):
- DM/no channel Ôćĺ friendly error "must be in a channel"
- Thread Ôćĺ use `thread.parent_id`
- Else Ôćĺ use `channel.id`

## Out of scope (future v1.14+ issues)

- `/skill info <name>` ÔÇö show full SKILL.md frontmatter + body excerpt (CLI doesn't expose, would need filesystem read)
- `/skill enable <name>` / `/skill disable <name>` ÔÇö uses `ClaudeAgentOptions.skills` field
- File watcher / auto-refresh ÔÇö `get_server_info()` is a session-startup snapshot, matches CLI behavior
- Pagination ÔÇö defer until a real skill load justifies it

## Rollback
