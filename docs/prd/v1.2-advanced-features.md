# v1.2 Advanced Features — Session Resume, Cost Tracking, Image Support

## Status
Approved

## Overview

3 个中等复杂度 feature，提升 claudeD 的持久性、成本可见性和多模态能力。

## Motivation

v1.1 解决了日常 UX，但三个痛点仍在：
1. Thread 被归档或 bot 重启后，session 丢失无法恢复——用户得重新建立上下文
2. 不知道花了多少钱，无法控制预算
3. 不能发图片给 Claude 分析

## Features

### F4 — Session Resume

**功能**：在 thread 中恢复之前的 Claude session，保留完整对话历史。

**实现**：
- `ClaudeBridge` 保存 `session_id`（从 SDK ResultMessage 获取）
- `SessionManager` 持久化 session 映射到 `data/sessions.json`：`{thread_id: {session_id, project_path, model, system_prompt}}`
- 新 thread 消息到达时，如果 `sessions.json` 有记录且没有活跃 bridge，用 `ClaudeCodeOptions(resume=session_id)` 恢复
- `/session resume` 命令：手动触发恢复
- Bot 重启后自动从 `sessions.json` 恢复映射（不预启动 bridge，lazy 恢复）

**接口**：
- `/session resume` — 手动恢复当前 thread 的 session
- 自动恢复：thread 中发消息时自动检测并恢复

### F5 — Cost Tracking

**功能**：追踪每个 channel/项目的累计花费，提供查询和预算控制。

**实现**：
- `CostTracker` 类（新文件 `cost_tracker.py`）：
  - `record(channel_id, cost_usd, model)` — 记录一次 API 调用费用
  - `get_channel_cost(channel_id) -> float` — 查询 channel 累计
  - `get_total_cost() -> float` — 全局累计
  - `reset(channel_id)` — 重置 channel 计数
  - 持久化到 `data/costs.json`
- `ClaudeBridge`：每次 `ResultMessage` 到达时，回调通知 cost
- `DiscordRenderer`：response 结束时在最后一条消息尾部加 cost 标注：`💰 $0.0234`
- 命令：
  - `/cost` — 显示当前 channel 累计花费
  - `/cost total` — 显示全局花费
  - `/cost reset` — 重置当前 channel

**接口**：`/cost [total|reset]`

### F11 — Image Support

**功能**：用户发送图片 → 传给 Claude 作为视觉输入。

**实现**：
- `bot.py` `_compose_user_text` 已有附件下载逻辑
- 扩展：识别图片文件（.png, .jpg, .jpeg, .gif, .webp）
- 图片路径传给 Claude：SDK 的 prompt 中可以引用本地文件路径，Claude Code 会读取
- 非图片附件保持现有逻辑（作为文件路径传递）
- 文本格式：`[User attached image: {filename}]\n{path}`

**接口**：无新命令，直接在消息中发图片即可。

## Acceptance Criteria

- [ ] AC1: Bot 重启后，在旧 thread 发消息能恢复之前的对话上下文
- [ ] AC2: `/session resume` 手动恢复 session
- [ ] AC3: `/cost` 显示当前 channel 花费
- [ ] AC4: `/cost total` 显示全局花费
- [ ] AC5: Response 结束时显示本次花费
- [ ] AC6: 发送图片后 Claude 能描述图片内容
- [ ] AC7: Cost 数据重启后不丢失

## Technical Approach

### F4 — Session Resume
```
data/sessions.json:
{
  "thread_id": {
    "session_id": "uuid",
    "project_path": "/path",
    "model": "sonnet",
    "system_prompt": "...",
    "last_active": "ISO timestamp"
  }
}
```
ClaudeCodeOptions(resume=session_id) 让 SDK 恢复历史。

### F5 — Cost Tracker
```
data/costs.json:
{
  "channels": {
    "123": {"total_usd": 1.23, "calls": 45},
  },
  "global_total_usd": 5.67
}
```

### F11 — Image Support
Claude Code SDK 接受文件路径。对图片文件，在 prompt 中标注为 image attachment，Claude 会用 Read 工具读取。

## Testing Strategy

- 单元测试：CostTracker 持久化、session resume 映射、图片文件类型检测
- Mock 测试：ClaudeBridge resume 参数传递

## Out of Scope

- 预算硬限制（告警但不阻断）
- 多用户费用分摊
- 视频/音频输入
- File diff 展示（v1.3）
- Claude 创建 channel（v1.3）
