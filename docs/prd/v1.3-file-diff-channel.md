# v1.3 File Diff Display + Claude Channel Management

## Status
Approved

## Overview

2 个 feature 完成 12 个 feature 的最后一批：文件变更可视化 + Claude 主动管理 Discord 结构。

## Features

### F7 — File Diff Display

**功能**：Claude 使用 Write/Edit 工具修改文件时，在 Discord 展示变更预览。

**实现**：
- `DiscordRenderer` 处理 `ToolUseBlock`（name=Write/Edit）和对应 `ToolResultBlock` 时：
  - Write 工具：显示文件名 + 内容预览（前 20 行 或 1500 字符）
  - Edit 工具：显示 old_text → new_text diff 格式
  - 用 Discord 代码块（```diff）展示
  - 超长内容截断 + "... (truncated)"
- 不改变现有工具状态提示（⚙️/✅/❌），在状态消息之后额外发一条 diff 预览

**接口**：无新命令，自动展示。

### F12 — Claude Channel/Thread Management

**功能**：让 Claude 能通过特殊指令创建 Discord channel 和 thread。

**实现**：
- 注入 `append_system_prompt` 告知 Claude 可用的特殊标记：
  - `[CREATE_THREAD: <name>]` — 在当前 channel 创建新 thread
  - `[CREATE_CHANNEL: <name>]` — 在当前 guild 创建新 text channel
- `DiscordRenderer` 解析 Claude 输出中的这些标记：
  - 检测到 `[CREATE_THREAD: xxx]` → 调用 Discord API 创建 thread
  - 检测到 `[CREATE_CHANNEL: xxx]` → 调用 Discord API 创建 channel
  - 替换标记为实际结果：`✅ Created thread: #xxx` 或 `❌ Failed: ...`
- 安全：只有 admin-bound 的 channel 中的 Claude 才能创建

**接口**：无新命令。Claude 在输出中使用标记。

## Acceptance Criteria

- [ ] AC2: Claude 执行 Write 工具时，Discord 显示文件内容预览
