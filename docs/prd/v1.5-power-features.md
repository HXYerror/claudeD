# v1.5 Power Features — Agents, MCP, Budget, Multi-Dir, Effort

## Status
Approved

## Overview

深度挖掘 Claude Code SDK 未利用的能力，将 claudeD 从简单的消息转发提升为完整的 Claude Code 控制台。

## Features

### F1 — Custom Agents (`/agent`)
Claude Code 支持自定义 agents（`--agents` 参数），每个 agent 有 description 和 prompt。

**功能**：
- `/agent create <name> <prompt>` — 创建自定义 agent（如 "reviewer", "translator"）
- `/agent list` — 列出可用 agents
- `/agent use <name>` — 在当前 thread 切换到指定 agent
- Agent 定义持久化到 `data/agents.json`
- ClaudeCodeOptions 中传 `extra_args={"agent": agent_name}` 或通过 agents JSON

**实现**：
- 新文件 `src/clauded/agent_manager.py`
- 修改 `claude_bridge.py` 传 agent 参数
- 新 slash command group `/agent`

### F2 — MCP Server 集成 (`/mcp`)
Claude Code 支持 MCP servers（`--mcp-config`），可以扩展 Claude 的工具集。

**功能**：
- `/mcp add <name> <command> [args]` — 为项目添加 MCP server（stdio 类型）
- `/mcp add-url <name> <url>` — 添加 HTTP/SSE 类型 MCP server
- `/mcp list` — 列出当前项目的 MCP servers
- `/mcp remove <name>` — 移除
- MCP 配置存入 project binding
- ClaudeCodeOptions 中传 `mcp_servers` 参数

**实现**：
- 扩展 `project_manager.py` 存储 mcp_servers 配置
- 修改 `claude_bridge.py` 传 mcp_servers
- 新 slash command group `/mcp`

### F3 — Budget Control (`/budget`)
Claude Code 支持 `--max-budget-usd` 参数。

**功能**：
- `/budget set <amount>` — 设置每个 session 的最大花费（美元）
- `/budget show` — 显示当前限制
- `/budget clear` — 移除限制
- 超出预算时 Claude 自动停止，bot 通知用户
- Budget 设置存入 project binding

**实现**：
- 扩展 `project_manager.py` 存储 budget
- 修改 `claude_bridge.py`，ClaudeCodeOptions 中传 `extra_args={"max-budget-usd": amount}`

### F4 — Multi-Directory Access (`/project add-dir`)
Claude Code 支持 `--add-dir` 让 Claude 访问额外目录。

**功能**：
- `/project add-dir <path>` — 添加额外目录访问权限
- `/project dirs` — 列出额外目录
- `/project remove-dir <path>` — 移除
- 额外目录持久化在 project binding 中

**实现**：
- 扩展 `project_manager.py` 存储 add_dirs 列表
- 修改 `claude_bridge.py`，ClaudeCodeOptions 中传 `add_dirs`

### F5 — Effort Level (`/effort`)
Claude Code 支持 `--effort` 参数（low, medium, high, xhigh, max）。

**功能**：
- `/effort <level>` — 设置 Claude 的思考深度
- 影响当前 thread 的 session
- 切换时重启 session

**实现**：
- 修改 `claude_bridge.py`，ClaudeCodeOptions 中传 `extra_args={"effort": level}`

### F6 — Allowed/Disallowed Tools (`/tools`)
Claude Code 支持 `--allowed-tools` 和 `--disallowed-tools`。

**功能**：
- `/tools allow <tool_names>` — 限制只用指定工具
- `/tools deny <tool_names>` — 禁用指定工具
- `/tools reset` — 恢复默认
- `/tools list` — 显示当前工具配置

**实现**：
- 修改 `claude_bridge.py`，ClaudeCodeOptions 中传 `allowed_tools` / `disallowed_tools`

### F7 — PR Review Mode (`/review`)
Claude Code 支持 `--from-pr` 参数。

**功能**：
- `/review <pr_url_or_number>` — 启动一个 PR review session
- 自动在当前 channel 创建 thread
- Claude 加载 PR context 并开始审查

**实现**：
- 新 slash command `/review`
- 修改 `claude_bridge.py`，ClaudeCodeOptions 中传 `extra_args={"from-pr": pr_ref}`

### F8 — Worktree Mode (`/worktree`)
Claude Code 支持 `--worktree` 自动创建 git worktree。

**功能**：
- `/worktree <name>` — 为当前 session 创建独立 worktree
- Claude 在隔离的 worktree 中工作，不影响主分支

**实现**：
- 修改 `claude_bridge.py`，ClaudeCodeOptions 中传 `extra_args={"worktree": name}`

### F9 — Session Fork (`/fork`)
Claude Code 支持 `--fork-session` 参数。

**功能**：
- `/fork` — 在当前 thread 基于当前 session 创建分叉
- 保留历史上下文但创建新 session ID
- 用于"从这里开始尝试不同方向"

**实现**：
- 使用 resume + fork-session 参数

### F10 — Compact/Context Management
Claude Code 有 `/compact` 和 `/context` 功能。

**功能**：
- `/compact` — 压缩当前 session 的上下文（减少 token 用量）
- 通过发送特殊消息触发 Claude 的内置压缩

**实现**：
- 发送 `/compact` 作为用户消息给 Claude session
