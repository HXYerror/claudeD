## Features to Implement

### F1 — Max Turns (`/max-turns`)
限制 Claude 每次回复的最大工具调用轮次。防止 Claude 死循环。
- `/max-turns <number>` 设置
- 传 `ClaudeCodeOptions(max_turns=N)`

### F2 — Fallback Model (`/fallback-model`)
当主模型过载时自动切换到备用模型。
- `/fallback-model <model>` 设置
- 传 `extra_args={"fallback-model": model}`

### F3 — Plan Mode 展示
Claude 进入/退出 Plan Mode 时在 Discord 展示状态。
- 检测 `EnterPlanMode`/`ExitPlanMode` ToolUseBlock
- 发蓝色 embed：`📋 Entered plan mode` / `✅ Exited plan mode`

### F4 — Task (子任务) 展示
Claude 启动子任务时展示。
- 检测 `Task` ToolUseBlock
- 发 embed 显示子任务描述
- `TaskOutput`/`TaskStop` 更新状态

### F5 — TodoWrite 展示
Claude 的 TODO 列表更新在 Discord 展示。
- 检测 `TodoWrite` ToolUseBlock
- 用 embed 展示 TODO 内容

### F6 — Hooks (PreToolUse 通知)
利用 SDK hooks 在工具执行前通知 Discord。
- 注册 `PreToolUse` hook
- 在 Discord 发状态消息（比现在的 ToolUseBlock 更早）

### F7 — Partial Messages (真 token 流式)
开启 `include_partial_messages`，实现真正的逐 token 打字机效果。
- 当前是 message 级别的流式（整个 TextBlock 一次）
- partial messages 是 token 级别
- 需要更新 DiscordRenderer 处理 StreamEvent

### F8 — Plugin Support (`/plugin`)
Claude Code 支持 plugin。
- `/plugin add <path>` — 添加 plugin 目录
- 传 `extra_args={"plugin-dir": path}`

### F9 — Security Review (`/security-review`)
映射 Claude 的 /security-review 内置命令。
- `/security-review` → 发送 "/security-review" 给 Claude session

### F10 — Settings Override (`/settings`)
允许传入自定义 settings JSON。
- `/settings <json>` — 传给 ClaudeCodeOptions(settings=json)

## Acceptance Criteria
- [ ] AC1-AC10: 每个 feature 对应的功能可用
